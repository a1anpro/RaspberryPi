#记录过程
目标：用google object detection api 来检测目标，用Opencv的跟踪算法来跟踪目标物
通过检测目标物周围的环境来分析发出警报，同时小车是跟着目标物移动的。

#1思路：将性能最好的kcf track和object detection一起使用，顺序执行，性能非常差
1的结果：google object detection api是用的ssd mobilenet，检测速度和kcf的跟踪速度不匹配
尝试更新一些优化。

#2思路：将kcf和detection封装，采用多线程的方式将速度较快的kcf track的执行时间分给detection的时间，这样可以在逻辑上
将顺序执行改成同步执行
